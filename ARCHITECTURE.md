## Service → Hook → Component Data Flow

```
┌─────────────┐     ┌──────────────┐     ┌───────────────┐     ┌────────────┐
│  mock-data   │ ──▶ │   service    │ ──▶ │  React Query  │ ──▶ │  Component │
│  (DELETE)    │     │  (SWAP HERE) │     │    hook       │     │    UI      │
└─────────────┘     └──────────────┘     └───────────────┘     └────────────┘
                           │
                     ┌─────▼──────┐
                     │  Prisma /  │  ← Future: replace mock imports
                     │  API fetch │
                     └────────────┘
```

**Key principle:** Hooks and components NEVER import from `src/data/` directly. Only services touch mock data, making the swap a single-file-per-module change.

## Prisma Schema Mapping

```prisma
// Matches src/types/index.ts exactly

model User {
  id        String   @id @default(cuid())
  name      String
  email     String   @unique
  role      UserRole
  team      Team
  active    Boolean  @default(true)
  joinedAt  DateTime
  avatar    String?

  acquisitionDeals Deal[]       @relation("AcqRep")
  dispoDeals       Deal[]       @relation("DispoRep")
  tcDeals          Deal[]       @relation("TC")
  kpiEntries       KpiEntry[]
  draws            Draw[]
  pointEvents      PointEvent[]
}

model Deal {
  id                    String     @id @default(cuid())
  propertyAddress       String
  sellerName            String
  buyerName             String?
  acquisitionsRep       User       @relation("AcqRep", fields: [acquisitionsRepId], references: [id])
  acquisitionsRepId     String
  dispoRep              User?      @relation("DispoRep", fields: [dispoRepId], references: [id])
  dispoRepId            String?
  transactionCoordinator User?     @relation("TC", fields: [transactionCoordinatorId], references: [id])
  transactionCoordinatorId String?
  contractDate          DateTime
  assignedDate          DateTime?
  closedFundedDate      DateTime?
  contractPrice         Float
  assignmentPrice       Float?
  assignmentFee         Float?
  buyerEmdAmount        Float?
  buyerEmdReceived      Boolean    @default(false)
  titleCompany          String
  inspectionEndDate     DateTime?
  status                DealStatus
  notes                 String     @default("")
  createdAt             DateTime   @default(now())
  updatedAt             DateTime   @updatedAt

  statusHistory DealStatusHistory[]
  draws         Draw[]
  pointEvents   PointEvent[]
}

model KpiEntry {
  id                String   @id @default(cuid())

  userId            String
  team              Team
  weekStarting      DateTime
  dials             Int
  talkTimeMinutes   Int
  leadsWorked       Int?
  offersMade        Int?

  falloutCount      Int      @default(0)

  buyerConversations  Int?
  propertiesMarketed  Int?
  emdsReceived        Int?
  assignmentsSecured  Int?
  avgAssignmentFee    Float?

model Draw {
  id              String     @id @default(cuid())
  rep             User       @relation(fields: [repId], references: [id])
  repId           String

  dealId          String

  dateIssued      DateTime

  notes           String     @default("")

  id                 String   @id @default(cuid())
```

## Migration Checklist for Cursor

1. **Scaffold Next.js** — `npx create-next-app@latest` with App Router + TypeScript
2. **Copy components/** — Drop in as-is (React components are framework-agnostic)
3. **Copy types/, data/, hooks/, services/, lib/** — No changes needed
4. **Convert pages/ → app/ routes** — Move page components into App Router file conventions
5. **Add `prisma/schema.prisma`** — Use the schema above
6. **Run `prisma migrate dev`** — Create tables
7. **Create `prisma/seed.ts`** — Port `src/data/mock-data.ts` into Prisma seed
8. **Swap services** — Replace mock imports with `prisma.deal.findMany()` etc.
9. **Add auth** — Clerk or NextAuth, protect routes with middleware
10. **Add API routes** — Wire form submissions to `/api/*` handlers
11. **Wire mutations** — Add `useMutation` hooks for create/update/delete
12. **Delete `src/data/`** — Remove all mock data files
